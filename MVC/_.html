Relations of Model

  1. cluster - collection of servers that communicate and work together for sync and redundancy
  2. server - a URL/machine that contains one or more databases
  3. database - collection of tables
  4. table - collection of rows/columns
  5. record - a row/collection of field/attributes
  6. field - data of type:
    float
    intiger
    money
    string
    html
    json

Directory Structure

The MVC code lives in the _lib git repository in a directory MVC. Each model has it's own directory with at files a:
m.mjs - contains model code
m?.mjs - Some modules will be broken up into helper modues. The main model code will create instances of m?
v.mjs - contains view code - simple display if model only, no direct user input. Implemented as a web component
v?.mjs - multiple views may be supported
c.mjs - control code that has user input and sends messages to model and view. c?msj - multiple controlers maybe supported Implemented as a web component _.cs - default style sheet for views and controls Demo - directory contains html files that demonstrate using MVC code for the given model
